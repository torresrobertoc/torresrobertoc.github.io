Case Study: Rincon Platform

TL;DR

I led the UX strategy for Rincon, a next-gen mortgage servicing platform. My work helped reduce training time by 80%, improve usability, and modernize the UI as we transitioned away from Visual FoxPro. The platform's UI redesign and improved user experience played a key role in DRI's acquisition by ServiceLink.

The Challenge:

Modernizing a Legacy System

Context:

  • The existing platform was built in Visual FoxPro and couldn't meet client demands for scalability or usability.
  • There were no dedicated UX resources—developers were designing the UI themselves.
  • Training and onboarding were time-consuming due to confusing layouts and terminology.

Constraints:

  • Needed to redesign while preserving complex business workflows and terminology.
  • Limited access to real users early on—we relied on internal SMEs and field interviews.

My Role & Responsibilities

I was a core contributor in the redesign of Rincon, working across UX, visual design, and front-end development. I introduced user-centered design practices, built a design system in XAML, and guided the project from research to implementation.

  1. User Research

    • Conducted on-site interviews with loan specialists and operations managers using the VFP platform.
    • Performed contextual inquiries to uncover workflow pain points.
    • Identified inconsistencies in terminology, layout, and field prioritization.
    • After the first client install, I again conducted a series of contextual interviews and generated a report that laid the foundation for the product roadmap: Download Field UX Report (PDF, 110KB)

  2. UI Design & Prototyping

    • Created wireframes and mockups for new workflows.
    • Redesigned information architecture to reduce cognitive load.
    • Delivered clickable prototypes for usability testing and stakeholder reviews.

  3. Visual Transformation & Systematization

      Here are a few selected screens from the Rincon redesign (later Fusion), showing the transition from legacy UIs to the redesigned interface. These views reflect my role in improving usability, visual hierarchy, and layout consistency.

  4. XAML UI Component Development

    • Defined and documented reusable components (e.g., form layouts, buttons, navigation).
    • Collaborated with the dev team to implement a scalable XAML-based UI pattern library.
    • Created developer guides, design principles, and implementation training materials.
    • This early XAML component work laid the foundation for the HTML/CSS-based Design System I later created for LoanSphere's modernized UI.

  5. Scaling the System Post-Acquisition

    • After I left DRI in 2011, it was acquired by ServiceLink and the Rincon platform was rebranded as Fusion. I was later recruited back as Sr. UI Engineer to formalize and scale the design system I had initiated.

The Solution:

A Modern UI with Workflow-Driven Design

  • User-Validated Layouts: Simplified the interface by prioritizing user actions over system data.
  • Terminology Alignment: Matched field labels and groupings to real-world workflows.
  • Design Documentation: Created principles and guidelines to ensure consistent development across teams — a precursor to the formal design system work I later led.
  • Visual Hierarchy: Used color, grouping, and alignment to reduce scanning effort and improve usability.

The Impact:

Quantifiable UX Improvements

  • ✅ 80% reduction in onboarding and training time.
  • ✅ Improved task completion rates for core user flows (validated through internal testing).
  • ✅ Higher developer efficiency through reusable XAML controls and better design documentation.
  • ✅ Contributed to acquisition value: The UI modernization and workflow improvements were highlighted as strategic assets during ServiceLink's acquisition of DRI.

Lessons Learned & Takeaways

  • Real Users Change Everything: Shadowing end users revealed issues our internal stakeholders never mentioned.
  • Consistency Reduces Friction: Design patterns and terminology reduced cognitive load across screens.
  • Documentation is Worth It: Developer-facing design notes accelerated and improved implementation.
  • Design ≠ Decoration: Good UX is about solving real workflow problems, not just making the UI pretty.
  • Systems Thinking Starts Early: Rincon's UI work helped shape my philosophy on scalable design, eventually leading to the enterprise-wide design system for LoanSphere.
