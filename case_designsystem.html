Case Study: Atomic Design System

TL;DR

I led the transition from a Silverlight (XAML) UI to a modern, scalable HTML/CSS-based UI framework. To support this shift, I built an Atomic Design System that ensured enterprise-wide UI consistency, reduced UI development time by 30%, and streamlined developer onboarding for the new Single-Page Architectures (SPA).

The Challenge:

Modernizing an Outdated UI Stack

Context:

  • The LoanSphere (formerly Fusion/Rincon) platform was transitioning away from Silverlight/XAML to a modern tech stack.
  • UI inconsistency was a major blocker—each team designed their own components, causing visual fragmentation and higher dev effort.
  • Accessibility (WCAG compliance) was not prioritized.
  • UI scalability was not planned, making the applications hard to maintain.

Constraints:

  • Teams were implementing different frameworks (React, Angular), so the system needed to be technology-agnostic, ensuring long-term viability and widespread adoptability.

My Role & Responsibilities

I played a foundational role in shaping the UI strategy for LoanSphere's transition to a modern stack, leveraging my expertise in UX, front-end development, and system design.

  1. Architectural Collaboration & Strategy

    • Engaged with architects, developers, and leadership to determine the best approach for modernizing the UI.
    • Advised on UI framework options (React, Angular), helping transition from Silverlight (XAML) to HTML, CSS, and JavaScript.
    • Built a browser-rendered, scalable system to ensure adaptability across evolving technology stacks.

  2. Design System Development

    • Established Atomic Design Principles to create consistent, reusable UI components.
    • Developed UI libraries using HTML, CSS, and jQuery, ensuring rapid implementation across projects.
    • Created a centralized documentation hub with developer notes and implementation guidelines.

  3. Developer Support & System Evangelism

    • Provided hands-on UI development support, guiding teams in adopting and implementing components.
    • Delivered training workshops and presented at virtual meetups to drive system adoption.
    • Maintained an iterative feedback loop, refining UI components based on real-world usage.

The Solution:

A Modular, Scalable Design System

  • Atomic Design Framework → Broke UI into Atoms, Molecules, and Organisms, ensuring consistency & scalability.
  • Component Library → Built reusable UI elements (buttons, forms, navigation, layouts) for faster development.
  • Developer Notes & Documentation → Created clear implementation guides to ensure devs understood the system.
  • Scalability & Flexibility → Designed for multiple applications & projects, reducing duplicate work.
  • Governance & Adoption → Established design principles & best practices for teams adopting the system.

Implementation in Action

  • First implemented in LoanSphere Loss Mitigation UI, then expanded to Bankruptcy PCN (100% adoption).
  • PCN became the first major implementation of the new design system UI components.
  • The design system was battle-tested, proving its scalability, efficiency, and flexibility in real-world application development.

Component in Action: Tile

To help streamline decision-making for loan specialists, I introduced a tile-based layout at the top of the Loss Mitigation interface. These tiles display critical borrower, loan, and property data at a glance—minimizing clicks and improving task flow.

<div class="tile">
    <header>Mortgagor(s)</header>
        <summary>
            <i class="fa fa-users fa-2x"></i>

Each tile used consistent semantic structure and styling for easy reuse and accessibility.

Design Highlights:

  • Reusable Molecule: Each tile is a modular UI component from the design system's Molecules tier.
  • Rapid Scan-ability: Prioritized typography and grouping for quick visual parsing of key data.
  • Consistency Across Screens: These tiles became a design pattern adopted in other modules.
  • Result: Reduced decision-making time and improved UX consistency across workflows.

The Impact:

Quantifiable UX & Dev Wins

  • ✅ 30% reduction in UI development time across projects.
  • ✅ Improved UI consistency across 5+ applications.
  • ✅ Faster onboarding for new developers and designers.
  • ✅ Reduced UX debt, preventing inconsistencies early.
  • ✅ Standardized accessibility & compliance, ensuring WCAG adherence.
  • ✅ Established a cross-team UI strategy, influencing future applications beyond my tenure.

Lessons Learned & Takeaways

  • Design Systems Require Evangelism - Developers don't adopt new systems overnight. Training, support, and advocacy were essential for success.
  • Documentation matters - Clear developer notes and example implementations ensured teams could implement the system with confidence.
  • Flexibility is Key - The system was designed to support multiple projects & future scalability.
  • Atomic Design scales well - Breaking UI into small reusable components led to efficiency.
